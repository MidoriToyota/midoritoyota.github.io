Market Basket Analysis - Parte 2¶



Esta a parte 2 (Criao do modelo preditivo) de um dos projetos de concluso do mdulo Business Analytics da Formao Cientista de Dados da Data Science Academy (DSA)

Definição do problema¶

Nesse projeto, utilizarei os dados annimos fornecidos pelo instacart para prever quais produtos adquiridos anteriormente estaro no prximo pedido de um usurio. O instacart um aplicativo de compras online, onde os compradores selecionam os produtos no app e a entrega feita na loja mais prxima voc.

The Instacart Online Grocery Shopping Dataset 2017, Accessed from:

https://www.instacart.com/datasets/grocery-shopping-2017 on 25/05/2020.

Post do VP da Instacart definindo o problema em detalhes:

https://tech.instacart.com/3-million-instacart-orders-open-sourced-d40d29ead6f2?gi=41f7b19cd164

Aplicação do modelo com threshold ideal¶

O threshold ser calculado com no ndice que fornecer a maior acurcia. Existem vrias formas de se calcular a acurcia, e aconfusionMatrix fornece uma delas. Essa acurcia calculada pela quantidade de acertos (positivos e negativos) dividido pelo nmero total da amostra. (TP + TN)/(P + N)

In [18]:
# Threshold ideal com base na acurcia dos resultados - pela frmula da confusion matrix
thresh = c(0)
acc = c(0)

for (i in 1:49) {
    # Formatao da tabela de sada
    rorder = rank(ypred, ties.method = "first" ) # Ranqueia valores ypred
    threshold = 0.02*i                              # 60% dos valores sero considerados "yes"
    ntop = length(rorder) - as.integer(threshold * length(rorder))
    plabel = ifelse(rorder > ntop, 1, 0)
    outdata_binario = list("prediction" = plabel) 

    # Matriz de confuso
    pred_xgboost = as.data.frame(outdata_binario)
    pred_xgboost$prediction = factor(pred_xgboost$prediction, labels = c("no", "yes"))
    data_test$reordered = factor(data_test$reordered, labels = c("no", "yes"))
    confusion = confusionMatrix(pred_xgboost$prediction, data_test$reordered)
    
    thresh[i] = threshold
    acc[i] = confusion$overall[1]
}

plot(x = thresh, y = acc, type = "l", ylab = "Accuracy", xlab = "Threshold", main = "Acurcia por threshold")
points(x = which.max(acc)*0.02, max(acc), col = "red", pch = 19,  cex = 2 )

Concluso : Mesmo que o threshold ideal tenha aumentado a acurcia de forma geral, ele aumenta a quantidade de falsos positivos, diminuindo assim a AUC. Isso acontece pois a frmula da acurcia, considera a acurcia total (true-positives e true-negatives) j a AUC considera o desempenho dos true-positives e false-positives.

Como o maior foco desse projeto descobrir os true-positives, levarei o resultado do threshold estimado para a concluso, pois ele demonstrou desempenho melhor na identificao de true-positives.

Vale destacar tambm, que muito mais prtico fazer uma estimativa, nem sempre ser vivel calcular precisamente qual o threshold ideal.

Essa concluso vlida apenas para esse problema. H momentos em que a acurcia total mais importante do que o valor da AUC, cabendo interpretao do analista.
