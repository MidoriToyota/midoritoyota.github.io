Codenation Challenge - Prevendo notas do Enem¶



Definição do problema¶

Voc dever criar um modelo para prever a nota da prova de matemtica de quem participou do ENEM 2016.

O contexto do desafio gira em torno dos resultados do ENEM 2016 (disponveis no arquivo train.csv). Este arquivo, e apenas ele, deve ser utilizado para todos os desafios. Qualquer dvida a respeito das colunas, consulte o Dicionrio dos Microdados do Enem 2016.

Muitas universidades brasileiras utilizam o ENEM para selecionar seus futuros alunos e alunas. Isto feito com uma mdia ponderada das notas das provas de matemtica, cincias da natureza, linguagens e cdigos, cincias humanas e redao, com os pesos abaixo:

  • matemtica: 3
  • cincias da natureza: 2
  • linguagens e cdigos: 1.5
  • cincias humanas: 1
  • redao: 3

No arquivo test.csv crie um modelo para prever nota da prova de matemtica (coluna NU_NOTA_MT) de quem participou do ENEM 2016. Salve sua resposta em um arquivo chamado answer.csv com duas colunas: NU_INSCRICAO e NU_NOTA_MT.

Qualquer dvida a respeito das colunas, consulte o Dicionrio dos Microdados do Enem 2016.

Tratamento de dados missing¶

Percebe-se que temos muitos dados nulos, inclusive na nossa varivel target. O quanto isso representa no nosso dataset? Vejamos abaixo:

In [8]:
# Funo - Proporo dos valores missing
propmiss = function(dataframe) {
  m = sapply(dataframe, function(x) {
    data.frame(
      nmiss = sum(is.na(x)),  
      n=length(x),
      propmiss = sum(is.na(x))/length(x) 
    )
  })
  d = data.frame(t(m))
  d = sapply(d, unlist)
  d = as.data.frame(d)
  d$variable = row.names(d)
  row.names(d) = NULL
  d = cbind(d[ncol(d)],d[-ncol(d)])
  return(d[order(d$propmiss), ])
}

Métricas¶

Para comparar os diversos modelos gerados, resolvi pegar o maior nmero de mtricas possvel. Vou utilizar as seguintes:

  • mae : Mean Absolute Error
  • mdae : Median Absolute Error
  • mse : Mean Squared Error
  • rae : Relative Absolute Error: sum(ae(actual, predicted)) / sum(ae(actual, mean(actual)))
  • rmse : Root Mean Squared Error
  • rrse : Root relative squared error
  • rse : Relative Squared Error : s sse(actual, predicted) / sse(actual, mean(actual))
  • sse : Sum of Squared Error
  • rsq : r-squared
In [215]:
# Funo para o clculo de r-squared (No consta no pacote Metrics)
rsq = function(x, y) summary(lm(y~x))$r.squared

# Criando um dataframe vazio para as mtricas
metricas = c("nome_incial",0,0,0,0,0,0,0,0,0)

# Funo para guardar as metricas de um modelo e retorn-las
calcula_metricas = function(modelo_nome, target_test, ypred) {
    
    # Mtricas de desempenho
    m1 = round(mae(target_test, ypred),2) # Mean Absolute Error
    m2 = round(mdae(target_test, ypred),2) # Median Absolute Error
    m3 = round(mse(target_test, ypred),2) # Mean Squared Error
    m4 = round(rae(target_test, ypred),2) # Relative Absolute Error: sum(ae(target_test, ypred)) / sum(ae(actual, mean(actual)))
    m5 = round(rmse(target_test, ypred),2) # Root Mean Squared Error
    m6 = round(rrse(target_test, ypred),2) # root relative squared error
    m7 = round(rse(target_test, ypred),2) # Relative Squared Error : s sse(target_test, ypred) / sse(actual, mean(actual))
    m8 = round(sse(target_test, ypred),0) # Sum of Squared Error
    m9 = round(rsq(target_test, ypred),2) # r-squared

    # Vetor de mtricas
    metricas_do_modelo = c(modelo_nome, m1, m2, m3, m4, m5, m6, m7, m8, m9)
        
    # Inserir as metricas no dataframe de mtricas
    metricas = rbind(metricas, metricas_do_modelo)
    colnames(metricas) = c("nome_modelo", "mae", "mdae", "mse", "rae", "rmse", "rrse", "rse", "sse", "rsq")
    metricas <<- metricas
    
    
    # Salvar as metricas para hisrico
    metricas_nome = paste("metricas_", modelo_nome)
    assign(metricas_nome, metricas_do_modelo, envir = .GlobalEnv)   
    }

O mae a mdia de todos os erros absolutos. Ou seja, a soma da diferena de cada valor actual menos o seu respectivo valor predicted (valores absolutos), dividido pelo nmero total de elementos. Sendo assim, uma forma de se medir o erro e quanto menor o erro, melhor o modelo. Pelo grfico percebemos que o XGBoost 2 teve o menor erro, seguido pelo Random Forest. J o modelo com pior desempenho nesse quesito foi o XGBoost 1.

O rmse a raiz quadrada da mdia dos erros ao quadrado. Tambm uma medida de erro e segue um padro semelhante ao mae.

O rsq, tambm chamado de coeficiente de determinao, uma medida estatstica de quo prximos os dados esto da linha de regresso ajustada, a porcentagem da variao da varivel resposta que explicada por um modelo e vai de 0 1. Sendo assim, uma medida de acurcia, e quanto maior o seu valor, melhor o modelo. No caso, novamente, O modelo do XGBoost mostrou-se melhor, seguido pela Random Forest, mas agora, os piores modelos foram os de Regresso Linear..
